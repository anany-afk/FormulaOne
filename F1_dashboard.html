// 02 — ARCHITECTURE

ANALYSIS PIPELINE

Five stages from raw session data to actionable insight. Each stage feeds directly into the next.

01
SESSION LOAD
FastF1 pulls official telemetry. All quick laps cached locally for speed.
02
SEGMENTATION
Every brake event is a boundary. Lap split into corner-level segments.
03
FEATURE ENG.
Speed, throttle, brake ratio, segment length extracted per segment.
04
RF CLASSIFIER
200-tree Random Forest predicts faster driver per segment. 80/20 split.
05
VISUALIZATION
9 plot types render the full picture — track maps, traces, cards, dashboard.
// 03 — OUTPUT

VISUALIZATION SUITE

Nine purpose-built plots. Each one answers a specific question about where time is won and lost.

TRACK MAP
SPEED HEATMAP
The actual GPS racing line colored by speed — red where the driver is scrubbing speed, green where they're carrying it. One glance tells you which corners are costing them.
📍
BRAKING
BRAKING ZONE MAP
Red dots on the track where each driver initiates braking. Per driver and head-to-head.
⏱️
DELTA
TIME LOSS MAP
Delta time between drivers painted onto the racing line. Exactly where time is gained or surrendered.
📏
BRAKING
BRAKING DISTANCE MARKERS
Who brakes later at each corner, annotated directly on the track map in metres.
📊
TELEMETRY
SYNCHRONIZED TRACES
Speed, throttle, and brake for both drivers overlaid in three synchronized panels, aligned by distance. Trail braking, early throttle, and lift points are all immediately visible.
🃏
CORNER
CORNER CARDS
One card per corner — braking point, min speed, exit throttle. Winner of each metric highlighted.
📈
CONSISTENCY
CONSISTENCY DASHBOARD
All quick laps overlaid with mean and ±1 std band. Tire degradation detrended before scoring. Verdict shows who is genuinely more consistent — not just who paced down more smoothly. Covers speed, throttle, and brake independently.
🤖
ML
MODEL VERDICT
Random Forest prediction plotted per segment — which driver the model picks as faster through each section.

// 05 — COMING NEXT

ROADMAP

Planned features in order of priority. The ML section is next.

🧬
DRIVER FINGERPRINTING
Unsupervised KMeans clustering on telemetry patterns. Can a model identify a driver purely from how they drive — with no labels?
🔍
ANOMALY DETECTION
Isolation Forest trained on normal laps. Automatically flags lock-ups, traffic, and mistakes — with the exact track location.
🧠
SHAP EXPLAINABILITY
Plug SHAP into the existing Random Forest. Shows not just who's faster per segment, but why — was it braking point, apex speed, or exit throttle?
🔥
OVERTAKING OPPORTUNITY MAP
Combine speed delta, DRS zones, and braking point data to score every track section for attack likelihood.
🏎️
OPTIMAL LAP RECONSTRUCTION
Stitch together the best sector from each lap across a stint to simulate the theoretical fastest possible lap.
📉
TIRE DEG MODELLING
Fit exponential regression to lap time progression. Predict when pace drops below a target threshold per compound.
